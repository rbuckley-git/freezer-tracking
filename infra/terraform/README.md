# AWS Terraform Infrastructure

This Terraform configuration provisions a simple AWS setup for `freezer-tracking`:

- VPC with 2 public subnets and 2 private subnets
- Internet gateway, NAT gateway, and public/private route tables
- ECS cluster running on a single EC2 instance (`t4g.small` by default)
- ECS services for API and web containers
- ECR repositories for API and web images
- Single internet-facing Application Load Balancer routing API and web services over HTTP/HTTPS
- Shared WAFv2 Web ACL attached to the public ALB using AWS managed rule groups and an IP rate limit rule
- Daily ECR housekeeping Lambda to remove images not used by active ECS task definitions
- Small Aurora PostgreSQL Serverless v2 cluster with one writer instance (`db.serverless`)
- Standalone PostgreSQL EC2 host in a private subnet with persistent EBS data volume and SSM management
- Generated database password stored in AWS Secrets Manager
- Generated API password pepper and web cookie key stored in AWS Secrets Manager

## Region

The default AWS region is `eu-west-2` and can be overridden with `aws_region`.

## Dependencies

Terraform and provider dependencies:

- `terraform` `~> 1.14.5`
- `hashicorp/aws` `= 6.27.0` — AWS resources for the main stack and bootstrap stack
- `hashicorp/archive` `~> 2.7` — package Lambda deployment artefacts in the main stack
- `hashicorp/random` `~> 3.7` — generate passwords, keys, and other random values in the main and bootstrap stacks

## Usage

1. Create a variables file from the example:

```bash
cp terraform.tfvars.example terraform.tfvars
```

2. Bootstrap remote state (one-time per AWS account/environment):

```bash
cd bootstrap
terraform init
terraform apply
terraform output
cd ..
```

3. Create backend config from the example and fill in the bootstrap outputs:

```bash
cp backend.hcl.example backend.hcl
```

4. Initialise and apply the main stack:

```bash
op run --env-file="./.aws.env" -- terraform init -backend-config=backend.hcl
op run --env-file="./.aws.env" -- terraform -chdir=infra/terraform apply
```

5. Optional variables:

- no mandatory variables

`api_image_tag` and `web_image_tag` select which tag ECS pulls from the Terraform-managed ECR repositories (default `latest`).
API and web services are always exposed through a shared internet-facing ALB using host-based routing.
The public ALB is configured as HTTPS-only (443). HTTP (80) is not exposed.
`alb_https_security_policy` controls the HTTPS security policy applied to public ALB HTTPS listeners.
`enable_alb_waf` controls whether a shared regional WAFv2 Web ACL is attached to the public ALB (default `true`).
`waf_rate_limit_per_5m` controls the per-IP request threshold for the WAF rate-based block rule (default `2000` over 5 minutes).
ECR housekeeping is always enabled with a scheduled Lambda that cleans old ECR images.
`ecr_housekeeping_schedule_expression` defaults to `rate(1 day)`.
`ecr_housekeeping_keep_unreferenced` keeps the newest unreferenced images per repo for rollback (default `2`).
`ecr_housekeeping_dry_run` logs candidates without deleting (default `false`).
`db_serverless_min_acu` and `db_serverless_max_acu` control Aurora Serverless v2 scaling bounds (defaults `0.5` and `2`).
`enable_rds_instance` controls whether Aurora compute instances are provisioned. Set to `false` to retain cluster storage/data while stopping Aurora compute billing.
`enable_rds_cluster` controls whether Aurora resources are provisioned at all. Set to `false` to remove Aurora cluster resources entirely.
`postgres_ec2_instance_type`, `postgres_ec2_data_volume_size_gb`, and `postgres_ec2_data_volume_type` control the standalone PostgreSQL EC2 host sizing.
`postgres_ec2_allowed_cidrs` can add extra client CIDR ranges for PostgreSQL access in addition to ECS tasks.
ECR repository policies are managed automatically and allow pulls for the Terraform-managed ECS task execution role and the current AWS account root principal (derived with `aws_caller_identity`).

## Notes

- Internet-facing ALBs and RDS subnet groups both require subnets in at least two availability zones, so this config creates two public and two private subnets even though the workload is intentionally single-instance/single-host.
- ALB health checks default to `/`; set `api_health_check_path` and `web_health_check_path` to dedicated endpoints if needed.
- ECS tasks use dynamic host ports with ALB target registration, which supports rolling deployments on the same ECS host when capacity allows.
- `api_health_check_path` and `web_health_check_path` are always used by their respective ALB target groups.
- The database password is generated by Terraform, written to Secrets Manager, used by Aurora PostgreSQL, and injected into the API task via ECS secrets.
- The same generated database password secret is also used to initialise the standalone PostgreSQL EC2 host, and the host is reachable via SSM Session Manager without SSH keys.
- `PASSWORD_PEPPER` (API) and `COOKIE_KEY` (web) are generated by Terraform, stored in Secrets Manager, and injected into ECS tasks via secrets.
- `web_api_base_url` can be used to override where the web container sends server-side API requests.
- The housekeeping Lambda evaluates images used by active ECS task definitions and deletes older unreferenced digests from the Terraform-managed API and web ECR repositories.
- Terraform state locking uses S3 lock files (`use_lockfile = true`).
- The default AWS VPC is adopted and its default network ACL is set to deny all ingress and egress traffic.

## ECR Push Example

After `terraform apply`, use the ECR output URLs to push images:

```bash
IMAGE_PLATFORM=linux/arm64 ./scripts/build-images.sh
./scripts/push-ecr-images.sh
```

If you prefer manual tagging/pushing:

```bash
aws ecr get-login-password --region eu-west-2 | podman login --username AWS --password-stdin <account_id>.dkr.ecr.eu-west-2.amazonaws.com
podman tag freezer-tracking-api:latest "$(terraform output -raw api_ecr_repository_url):latest"
podman push "$(terraform output -raw api_ecr_repository_url):latest"
podman tag freezer-tracking-web:latest "$(terraform output -raw web_ecr_repository_url):latest"
podman push "$(terraform output -raw web_ecr_repository_url):latest"
```

## Bootstrap Admin Account On ECS

Use a one-off ECS task with environment overrides to create an initial admin account safely and idempotently.
This requires API image `2.0.0` or later.

```bash
aws ecs run-task \
  --cluster freezer-tracking-cluster \
  --launch-type EC2 \
  --task-definition freezer-tracking-api-bootstrap \
  --count 1 \
  --overrides '{
    "containerOverrides": [{
      "name": "api",
      "environment": [
        {"name":"BOOTSTRAP_ADMIN_ENABLED","value":"true"},
        {"name":"BOOTSTRAP_ADMIN_EXIT_AFTER_RUN","value":"true"},
        {"name":"BOOTSTRAP_ADMIN_USERNAME","value":"admin@example.com"},
        {"name":"BOOTSTRAP_ADMIN_PASSWORD","value":"replace-with-strong-password"},
        {"name":"BOOTSTRAP_ADMIN_HOUSE_NAME","value":"Primary House"}
      ]
    }]
  }'
```

Expected behaviour:
- If the account does not exist, it is created as admin.
- If the account already exists, it is promoted to admin and assigned to the configured house.
- The task exits after the bootstrap run when `BOOTSTRAP_ADMIN_EXIT_AFTER_RUN=true`.

## Cloudflare DNS and HTTPS

1. Apply Terraform to request ACM certificates and create the shared HTTPS-only public ALB.
2. Read `cloudflare_acm_validation_cnames` from Terraform outputs and create those CNAME records in Cloudflare.
3. Wait for ACM status to become `ISSUED`.
4. Read `cloudflare_service_cnames` and create those CNAME records in Cloudflare:
   - `freezer.learnsharegrow.io` -> web ALB DNS name
   - `freezer-api.learnsharegrow.io` -> API ALB DNS name
5. Keep the ALB HTTPS-only and complete DNS cutover once certificate validation is issued.
